# Competency Governance v2
역량 라이브러리(Competency) 및 스킬(Skill) 관리와 운영 규칙을 정의하는 문서.

## 1. 데이터 등록 원칙
- **재사용 우선**: 관리자 및 발주처는 가급적 시스템에 등록된 기존 핵심 역량을 재사용하는 것을 권장.
- **의미의 보편성 유지**: 역량 정의는 특정 직무나 도구에 종속되지 않고 범용적으로 적용될 수 있는 심리/행동적 용어로 서술.

## 2. 권한
- `skills` 및 `competencies` 테이블은 관리자 계정(Operator)만이 신규 등록 및 수정 권한을 가짐(RLS 설정).
- 일반 사용자는 목록에서 '선택'만 가능.

## 3. 수정 및 버전 관리 정책
- **단순 가이드 수정(의미 확장)**: 역량의 Description, 예시 추가 등 평가 기준에 타격이 없는 수정은 "수정"으로 기존 테이블 직접 업데이트 가능. (이전 경험에 소급 적용해도 문제가 없음)
- **체크리스트/레벨 구조 변경**: 체크리스트 문구의 본질적 변경(Yes/No 결정에 영향을 미칠 정도), 레벨 앵커 문구 교체 등 "평가 기준" 변경 시 기존 경험 모델 보존을 위해 새 버전을 추가해야 함 (`competency_rubric_versions`의 version 상승).
- 이전 버전에 연결된 사용자 경험(`experience_competencies`)은 스냅샷을 캡처하여 생성 당시의 버전을 영구 보존.

## 4. 데이터 삭제 원칙 (금지)
- **삭제 불가**: 한 번 등록되어 1건이라도 사용자의 경험에 연결된 Skill/Competency는 DB에서 `DELETE` 금지.
- **Soft-Delete(비활성화) 사용**: 사용 자제를 위해 `is_active=false`로 설정. 향후 신규 입력에서는 노출되지 않지만 기존에 기록한 데이터 통계나 조회에는 정상 노출된다.
